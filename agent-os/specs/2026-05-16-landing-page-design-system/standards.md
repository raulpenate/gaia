# Standards for Landing Page + Design System

The following standards apply to this work:

- `frontend/tailwind-usage` -- @theme tokens, utility-first, no @apply in components
- `frontend/component-structure` -- one component per file, PascalCase, co-locate types
- `frontend/accessibility` -- WCAG 2.1 AA, semantic HTML, keyboard nav, focus management
- `nextjs/server-first` -- server components by default, "use client" only at interactive leaves
- `i18n/message-namespaces` -- Landing namespace, every key in both locale files
